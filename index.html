📋 プロジェクト概要

このプロジェクトは何？

eBayで商品を販売するときに必要な「分析」と「リサーチ」を、AIの力を借りて効率化するプロジェクトです。

今まで目で見て、手作業でやっていたことを、AIと自動化ツールで楽にすることが目標です。

解決したい問題

😫 今の状態（Before）

  • ❌ 検索結果を全ページ目視で確認
  • ❌ 新しいキーワードの発掘が大変
  • ❌ 分析とリサーチがつながっていない
  • ❌ 毎日の出品記録が面倒
  • ❌ データがバラバラで判断しにくい

😊 目指す状態（After）

  • ✅ AIが検索結果を自動で分析
  • ✅ 売れ筋から新キーワードを提案
  • ✅ 分析→リサーチ→出品が一連の流れに
  • ✅ 出品データは自動で記録
  • ✅ データが一箇所に集まり判断しやすい

⚠️ ボトルネック（一番の問題点）

「分析」から「リサーチ」への橋渡しがない

せっかくぶんせき君で良いデータが取れても、そこからのリサーチ作業が手作業のため、時間がかかりすぎている。検索結果を1ページずつ見ていくのは人間の限界を超えている。

🎯 目標

短期目標（1-2週間）

  • MCP（AIの拡張機能）のセットアップを完了する
  • 毎日の出品データを自動で記録する仕組みを作る
  • データの保存先を決めて統一する

中期目標（1-2ヶ月）

  • 分析結果から自動でリサーチキーワードを提案する
  • 競合セラーの動向を自動で追跡する
  • 週次レポートを自動生成する

長期目標（3ヶ月以降）

  • AIエージェントとして、相談しながら出品戦略を決められる
  • カテゴリの強化・撤退判断をデータに基づいて行う
  • 新規カテゴリ開拓を半自動化する

✨ 最終的に目指す姿

毎朝、AIが「今日やるべきこと」を提案してくれる。例えば：

  • 「昨日出品した○○カテゴリ、競合より20%安いので価格調整を検討してください」
  • 「△△ブランドが今週急上昇中です。関連商品のリサーチをおすすめします」
  • 「□□カテゴリは3週間売上ゼロです。撤退を検討しますか？」

各層の役割

📥 データ収集層

eBayから様々なデータを集めてくる部分です。

  • ぶんせき君：自分の出品データ、Terapeakの市場データ、競合セラーのデータを取得
  • Webスクレイピング：将来的に、他のサイトからも情報を自動取得

💾 データ蓄積層

集めたデータを整理して保存する場所です。

  • Google Sheetsを中心に、時系列でデータを蓄積
  • 過去のデータと比較できるようにする
  • グラフや表で可視化しやすい

🤖 分析・AI層

集めたデータをAIが分析して、提案を出す部分です。

  • Claude Code：メインのAIアシスタント
  • MCP：AIに様々なツールを使わせるための仕組み

📊 アウトプット層

AIが分析した結果を、実際のアクションにつなげる部分です。

  • 今日やるべきことのリスト
  • 新しく調べるべきキーワード
  • カテゴリの強化や撤退の判断材料

🔌 MCP（AI拡張機能）一覧

💡 MCPって何？

MCP（Model Context Protocol）は、AIに様々なツールを使わせるための仕組みです。

例えるなら、スマホにアプリを入れるようなもの。MCPを入れることで、AIができることが増えます。

USBポートのように、いろんなツールを「接続」できる標準規格です。

日次ワークフロー（毎日やること）

🌅 朝：前日の振り返り

  • 前日の出品数・売上を確認
  • 新規注文をチェック
  • AIから「今日のおすすめアクション」を受け取る

🌞 日中：作業記録

  • 出品した商品を記録（自動化が目標）
  • リサーチで見つけたキーワードをメモ

🌙 夜：その日のまとめ

  • 作業内容を登録
  • 明日やることをメモ

週次ワークフロー（週1回やること）

📊 週次レビュー

  • 今週の出品数 vs 売上の振り返り
  • カテゴリ別のパフォーマンス確認
  • 競合セラーの動向チェック
  • 来週の強化カテゴリを決定

月次ワークフロー（月1回やること）

📈 月次分析・戦略会議

  • 月間の売上・利益の分析
  • カテゴリ別の継続/撤退判断
  • 新規カテゴリ開拓のリサーチ
  • 来月の目標設定

📚 用語集

MCP（Model Context Protocol）

AIに様々なツールを使わせるための仕組み。Anthropic社が2024年に発表。「AIのUSBポート」のようなもので、いろんなツールを接続できる。

Claude Code

Anthropic社が提供するAIアシスタント「Claude」のコマンドライン版。ターミナルで動作し、MCPを通じて様々なツールと連携できる。

Terapeak

eBayが提供する市場調査ツール。売れ筋商品、価格帯、競合の情報などを調べられる。eBayセラーハブから利用可能。

GAS（Google Apps Script）

Googleが提供するスクリプト言語。スプレッドシートの自動化などに使える。JavaScriptベースで比較的学びやすい。

n8n

ワークフロー自動化ツール。「もしAが起きたらBをする」という自動化を、プログラミングなしで作れる。無料のセルフホスト版と有料のクラウド版がある。

Webスクレイピング

Webサイトから自動的にデータを取得する技術。例えば、競合の出品情報を自動で集めることができる。

API（Application Programming Interface）

ソフトウェア同士が情報をやり取りするための窓口。例えばeBay APIを使うと、プログラムからeBayのデータを取得できる。

APIキー

APIを使うための「鍵」。サービスに登録すると発行される。この鍵がないとAPIは使えない。
